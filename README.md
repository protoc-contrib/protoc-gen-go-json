A [protoc](https://protobuf.dev) plugin that emits `MarshalJSON` and
`UnmarshalJSON` methods for Protocol Buffer messages. The generated methods
delegate to
[`google.golang.org/protobuf/encoding/protojson`](https://pkg.go.dev/google.golang.org/protobuf/encoding/protojson),
so any message wrapped in an `encoding/json` pipeline gets proto-faithful
output and parsing — including correct handling of oneofs, maps, enums, and
`google.protobuf.*` well-known types — without callers having to reach for
`protojson` directly.

This project is a fork of
[mitchellh/protoc-gen-go-json](https://github.com/mitchellh/protoc-gen-go-json).
The code generator has been rewritten on top of
`google.golang.org/protobuf/compiler/protogen` (replacing the previous
`text/template`-based renderer), the layout now mirrors the rest of the
`protoc-contrib` plugins, and the build and release pipeline runs on Nix +
`release-please`.

## Features

- `json.Marshaler` and `json.Unmarshaler` for every message declared in your
  `.proto` files, including nested messages.
- Transparent interop with `encoding/json` — wrap a proto message in any
  other Go struct and the JSON is still canonical.
- Configurable protojson behavior via plugin options (enum rendering, zero
  values, original field names, tolerance for unknown fields).
- Respects proto3 optional presence.

## Installation

```bash
go install github.com/protoc-contrib/protoc-gen-go-json/cmd/protoc-gen-go-json@latest
```

## Usage

### With buf

Add the plugin to your `buf.gen.yaml`:

```yaml
version: v2
plugins:
  - remote: buf.build/protocolbuffers/go
    out: gen/proto/go
    opt:
      - paths=source_relative
  - local: protoc-gen-go-json
    out: gen/proto/go
    opt:
      - paths=source_relative
```

Then run:

```bash
buf generate
```

### With protoc

```bash
protoc \
  --go_out=. --go_opt=paths=source_relative \
  --go-json_out=. --go-json_opt=paths=source_relative \
  -I proto/ \
  proto/example/v1/example.proto
```

## Options

All options are booleans and are forwarded to `protojson.MarshalOptions` /
`protojson.UnmarshalOptions`.

| Option          | Default | Maps to                                   | Purpose                                                                |
| --------------- | ------- | ----------------------------------------- | ---------------------------------------------------------------------- |
| `enums_as_ints` | `false` | `MarshalOptions.UseEnumNumbers`           | Render enums as integers instead of their string names.                |
| `emit_defaults` | `false` | `MarshalOptions.EmitUnpopulated`          | Emit fields whose value is the zero value for their type.              |
| `orig_name`     | `false` | `MarshalOptions.UseProtoNames`            | Use the original `.proto` field names instead of `lowerCamelCase`.     |
| `allow_unknown` | `false` | `UnmarshalOptions.DiscardUnknown`         | Ignore unknown fields during unmarshaling instead of returning errors. |

Pass them through `buf.gen.yaml`:

```yaml
plugins:
  - local: protoc-gen-go-json
    out: gen/proto/go
    opt:
      - paths=source_relative
      - emit_defaults=true
      - orig_name=true
```

or on the protoc command line:

```bash
protoc --go-json_opt=emit_defaults=true,orig_name=true ...
```

## Example

```proto
syntax = "proto3";
package example;

message User {
  string id = 1;
  oneof contact {
    string email = 2;
    string phone = 3;
  }
  map<string, string> labels = 4;
  optional string nickname = 5;
}
```

Generated alongside `example.pb.go`:

```go
// Code generated by protoc-gen-go-json. DO NOT EDIT.
// source: example.proto

package example

import (
    protojson "google.golang.org/protobuf/encoding/protojson"
)

func (msg *User) MarshalJSON() ([]byte, error) {
    return protojson.MarshalOptions{
        UseEnumNumbers:  false,
        EmitUnpopulated: false,
        UseProtoNames:   false,
    }.Marshal(msg)
}

func (msg *User) UnmarshalJSON(b []byte) error {
    return protojson.UnmarshalOptions{
        DiscardUnknown: false,
    }.Unmarshal(b, msg)
}
```

## CI Integration

Gate builds on the generated `*.pb.json.go` files being up-to-date by running
`buf generate` in CI and failing if the worktree is dirty.
